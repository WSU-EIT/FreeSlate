Headings & header rows

Slate's Heading 1, Heading 2, and Header Row field types. Use exactly one H1 per form (the form title), then H2s for sections — never skip levels. The crimson rule under the H1 is automatic.

Text box

The workhorse. One field = one label + one input, with optional help text tied by aria-describedby. Add wsuf-field--required for the crimson required edge.

Slate: Text Box ✓ label[for] + describedby

Enter the full name, e.g. "Pullman High School".

<div class="wsuf-field wsuf-field--required">
  <label for="school">Current or most recent school name</label>
  <p class="wsuf-help" id="school-help">Enter the full name.</p>
  <input type="text" id="school" aria-describedby="school-help" required>
</div>

Email, phone & number

Same field pattern, smarter type. The right type triggers the right mobile keyboard and built-in format checks — and helps screen readers and autofill. Always set autocomplete.

type=email · tel · number ✓ inputmode + autocomplete
<input type="email" id="email" autocomplete="email" required>
<input type="tel"   id="phone" autocomplete="tel" inputmode="tel">
<input type="number" id="credits" min="0" max="200">

Choices

Option buttons (radio)

Pick exactly one. Always wrap a radio set in a <fieldset> with a <legend> — that legend is the group's question, and it's what makes the group make sense to a screen reader. Use the card variant for big, important choices.

Slate: Option Buttons — plain ✓ fieldset + legend
Will you have transfer credit from AP, IB, or Running Start? 
<fieldset>
  <legend>Will you have transfer credit?</legend>
  <div class="wsuf-choices">
    <div class="wsuf-choice">
      <input type="radio" id="tc-y" name="tc"><label for="tc-y">Yes</label>
    </div>
    <div class="wsuf-choice">
      <input type="radio" id="tc-n" name="tc"><label for="tc-n">No</label>
    </div>
  </div>
</fieldset>

Date & place

Date & birthdate

⚠ Slate gotcha

Slate's jQuery-UI datepicker opens under the sticky header (z-index). The kit ships the fix — .ui-datepicker{z-index:200 !important}, one of only two sanctioned !importants. Also never bundle a second jQuery: Slate loads its own, and a duplicate breaks the datepicker and conditional logic.

This is where your current form loses the most WAVE points. Slate's classic Date/Birthdate emits three separate month/day/year selects — and the day and year ones come out with empty labels. A native type="date" (or type="month") is one labeled control: zero empty labels, a built-in calendar, and full keyboard support.

type=date / type=month ✓ one control, one label
<!-- One control. No empty month/day/year selects. -->
<div class="wsuf-field wsuf-field--required">
  <label for="bd">Student birthdate</label>
  <input type="date" id="bd" required>
</div>

If you must keep Slate's three-select date, the fix is to give the day and year selects a real label or aria-label ("Day", "Year") — that alone clears those empty-label errors. But prefer the native control.

Street address

♿ Accessibility

Two traps. (1) Sub-field labels aren't editable in the builder — add the required asterisk via CSS ::after on the emitted labels (see “Slate Defaults, Fixed”). (2) Changing Country AJAX-swaps the State select, wiping any JS-injected aria-labels — re-apply with a MutationObserver and announce via a polite live region.

Slate's Street Address composite. The second WAVE offender — its sub-fields often render unlabeled. Give every part its own label, group them in a fieldset, and set the right autocomplete tokens so browser autofill works.

Slate: Street Address ✓ every part labeled

Signature

♿ Accessibility

A canvas signature is unusable by keyboard and screen-reader users. Always pair it with a typed-full-name + checkbox attestation alternative (below), give the canvas role="img" + a label, and never make the drawn signature the only accepted path — it's legally safer too.

Slate's Signature field. The pad needs an accessible label and a typed-name fallback for people who can't draw with a mouse/finger.

Slate-managed

Fields Slate renders itself

⚠ Slate gotcha

The Translation / Google Translate widget rewrites text nodes and wraps content in <font> tags — never hang styling off text-content selectors, ::first-letter, or exact-text attribute selectors; they break silently for translated users. Widget Table inlines its own column widths, so overrides need equal specificity or CSS custom properties. Keep <html lang> correct.

A few field types are driven by Slate's own server-side widgets — Related Event, Widget Table, Interaction Selection, Video Capture, and Translation. You generally can't change their internal markup, but the kit's base styles (inputs, buttons, tables, labels) flow into them, so they inherit the WSU look automatically. The rule for these: make sure whatever Slate emits has a visible, associated label, and don't remove focus outlines.

Related Event

Event/session picker. Inherits select & radio styling.

Widget Table

Tabular input. Kit styles table.table with crimson header rules & zebra rows.

Interaction Selection

Links a Slate interaction record. Renders as a styled select.

Video Capture

Records a short video. Style the surrounding label/button only.

Translation

Localized field variants. No special styling needed.
