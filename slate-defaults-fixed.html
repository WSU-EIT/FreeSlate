Slate Defaults, Fixed

The papercuts every Slate shop hits — the ugly or inaccessible defaults Slate ships — and the exact CSS or setting that fixes each one. Most are already handled by the kit's mapping layer; this page is the receipts.

CSS = already in the kit UI = a Slate setting

Fixes

  • Address-block labels
  • 3-select date
  • Submit button
  • Rating-scale labels
  • Datepicker colors
  • Address country default
  • Logged-in bar
  • Required marker

↩ Kit

  • Back to home

Each card: the annoying default, where it's fixed (CSS = automatic once the kit is deployed; UI = a setting you click), and the copy-paste snippet. Two items are flagged VERIFY — confirm the exact Slate class against a rendered form before relying on them.

1

Required asterisk won't show on address-block labels

Slate's Street Address composite doesn't let you mark sub-fields required in the editor, so no asterisk appears.

CSS · in the kit
/* Force the asterisk + crimson edge on required address parts.
   Already shipped if you mark the address field required in Slate. */
.wsu-content .form_question[data-required="1"] .form_label::after {
  content: " *"; color: var(--wsu-red); font-weight: 700;
}

VERIFY: confirm Slate tags the required question with [data-required="1"] (vs .form_required) in your instance.

2

3-dropdown date emits empty labels (24 WAVE errors)

Birthdate/Date as month + day + year selects leaves the day and year unlabeled.

UI · field setting
<!-- Prefer a single native control instead of the 3-select widget -->
<div class="wsuf-field wsuf-field--required">
  <label for="bd">Student birthdate</label>
  <input type="date" id="bd" required>
</div>

If you must keep the 3-select widget, add aria-label="Day" / aria-label="Year" to the unlabeled selects via the form's Edit Scripts.

3

Submit button is unbranded and says "Submit"

Stock Slate submit is a plain gray/blue button with generic text.

CSS + UI
/* The kit already styles the submit (crimson, Montserrat, hover).
   To RENAME it: form editor → Properties → Submit Button Text. */
/* To restyle a specific form's submit only: */
#form_GUID_container button[type="submit"] { font-size: 1.15rem; }

Text is a UI setting (Properties → Submit Button Text, e.g. "Submit registration"); styling is automatic via the kit.

4

Rating scale shows redundant per-point labels

Slate repeats a label under every point; you usually only want labeled endpoints.

CSS · in the kit
<!-- Use the kit rating pattern: numbers + labeled ends only -->
<div class="wsuf-rating"> … 1–5 … </div>
<div class="wsuf-rating__ends"><span>Not interested</span><span>Very interested</span></div>

See the Field Type Library → Rating Scale for the full markup.

5

Scheduler datepicker uses default (non-brand) colors

Slate's jQuery-UI calendar marks available days in its own palette.

CSS · in the kit
/* Already in the kit's Slate mapping layer */
.wsu-content .ui-datepicker-calendar .available a {
  color: var(--wsu-crimson-ink); border: 1px solid var(--wsu-crimson);
}
.calendar_legend.available { background-color: var(--wsu-crimson) !important; }
6

Address country defaults blank (extra click for 99% of users)

Most applicants are US-based; defaulting saves everyone a step.

UI · field setting
In the form builder, open the Street Address field → set the Country default to United States. No CSS needed. The kit then shows the US state dropdown by default.
7

The gray "logged in as…" identity strip is unbranded

Slate prepends a plain gray bar with the user's name + Log Out.

CSS · in the kit
/* Kit restyles Slate's emitted div[role="banner"] to WSU crimson */
.wsu-content > div[role="banner"] {
  background: var(--wsu-crimson); color: #fff;
  padding: .5rem 1.5rem; display: flex; justify-content: space-between;
}

Automatic — see States & Edge Cases for the before/after.

8

Required fields aren't visually obvious

Stock Slate gives little visual signal that a field is required.

CSS · in the kit
/* WDS convention: crimson left edge + asterisk on required controls.
   Automatic once you mark the field required in Slate. */
.wsu-content .form_question[data-required="1"] input { border-left: 3px solid var(--wsu-crimson); }

VERIFY the required attribute/class, then ship. Add a "Fields marked * are required" note at the top of every form.
