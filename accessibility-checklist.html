Labels & names

Every control has an accessible name

WAVE "Missing form label" and "Empty form label" are the top Slate error sources. Follow these items to eliminate them all.

Never leave a label empty. Even a "hidden" label for layout must contain text — use .wsuf-sr-only if visual label isn't needed.

WAVE: Empty label

A group of related choices needs a shared question. <fieldset><legend>Question?</legend>…</fieldset>

WAVE: Missing label

The three-dropdown widget emits unlabeled month/day/year selects. Replace with a single native date control. Your current form scores 24 empty-label WAVE errors from this alone.

⚠ Slate trap: Birthdate, Date, and Address fields are the #1 source of empty-label errors.
WAVE: Empty label ×3

<input aria-describedby="field-help"> and <p id="field-help">. Screen readers read both label and description.

WCAG 1.3.1

When a field has an error: set aria-invalid="true" on the input and add the error ID to aria-describedby.

WAVE: Broken aria-reference

SVG icons: use role="img" aria-label="..." + <title>. Decorative SVGs: aria-hidden="true".

WAVE: Missing alt
Keyboard & focus

Every interaction is keyboard-operable

Nothing hidden without display:none/visibility:hidden/inert. Custom widgets need tabindex="0" and keyboard handlers.

WCAG 2.1

The kit provides a crimson focus ring via :focus-visible. Never add outline: 0 without a custom focus indicator. Test with keyboard, not mouse.

WCAG 2.4.11 AAA

The kit sets min-height: 2.75rem (44px) on choice wrappers. Verify no override shrinks them.

WCAG 2.5.8

The kit provides <a class="wsu-skip" href="#main">. Confirm it appears visually on first Tab press and that the target has tabindex="-1".

WCAG 2.4.1

When revealing hidden content with JS (e.g. pronouns self-describe), focus the first newly-visible control. Hidden fields use hidden attribute — not just visibility:hidden.

WCAG 2.1.1
Color & contrast

Text has sufficient contrast

Automatic with the kit. Don't change body text color below #595959 (7:1 AAA).

AAA

Kit uses --wsu-muted: #595959 for help text. Lighter grays will fail.

AAA

Kit buttons use crimson default (AAA). Don't switch to Bright Red #ff0843 — white on that is only 3.3:1 and fails AA.

AAA

Crimson on #333 = 1.64:1 — catastrophic fail. On dark surfaces (header, footer, banner), always use white text.

⚠ WSU trap: The real admission.wsu.edu has 12+ WAVE contrast errors from exactly this pattern.
WAVE: Very low contrast

Kit error messages use a "!" icon, red text, and aria-invalid. Validation state is never conveyed by border color alone.

WCAG 1.4.1
Structure

The page has correct landmarks

Section headings are H2. Never use H1 more than once. Slate section fields should output H2.

WAVE: Skipped level

WAVE flags heading-level skips as structural errors. Don't use H3 without an H2 above it in the same section.

WAVE: Skipped heading

Multiple banner/main landmarks confuse screen reader users. Slate's build.xslt provides these — don't add extra ones in form content.

WCAG 1.3.6

<nav aria-label="WSU utility">, <nav aria-label="Breadcrumb">, <nav aria-label="Form sections">. Unique labels let screen reader users distinguish them.

WAVE: Duplicate landmark
Error handling

Validation is accessible

Kit: <div class="wsuf-alert" role="alert" tabindex="-1">. On failed submit, call .focus() on the summary. Screen readers announce role=alert automatically.

WCAG 3.3.1

Each error list item links to the field ID: <a href="#email">Email — required</a>. Clicking moves focus to the field, not just scrolls.

WCAG 3.3.3

<div role="status" tabindex="-1">. Call .focus() on it after submit. role=status is polite (doesn't interrupt ongoing speech).

WCAG 4.1.3

Kit: wsuf-field--required adds the asterisk; the actual HTML required attribute is the machine-readable signal. Note the required convention at top of form.

WCAG 1.3.5
Slate-specific

Known Slate traps

These are the patterns Slate generates that will fail WAVE unless you specifically override them.

Slate's Birthdate widget produces Month (labeled), Day (empty label), Year (empty label) = 2 WAVE errors per field. Use HTML date input pattern from the Field Type Library.

24 errors on your form

Slate's Street Address often omits labels for City, State, ZIP. Wrap in a fieldset, give every sub-field a label, set autocomplete attributes.

WAVE: Empty labels

Build.xslt noscript iframes should have style="display:none;visibility:hidden". WAVE flags visible zero-size frames as alerts.

WAVE: Noscript element

The "×" button on multi-select chips must say what it removes: aria-label="Remove Applied Mathematics". A bare "×" gives screen readers nothing.

WAVE: Empty button
Advanced AAA

AAA & zero-warning targets

The harder criteria from the accessibility standard — these take you from "AA clean" to "AAA, zero WAVE alerts."

The kit's --wsu-control-border fixes the WDS default that failed 1.4.11. Don't lighten control borders.

WCAG 1.4.11

Box-shadow focus rings vanish in forced-colors. The kit's @media (forced-colors:active) swaps in a solid outline — verify badges/banners survive with system colors.

1.4.1 / system

Test the event two-column layout and any grid collapse at 320px.

WCAG 1.4.10

Badges/chips use min-height, not fixed height, so increased line-height/spacing doesn't clip them.

WCAG 1.4.12

given-name, family-name, email, tel, the address-line set, bday. Slate-generated fields that lack it get it via the delegated-JS layer.

WCAG 1.3.5

Typed-name + attestation for Signature; file upload for Video Capture. Never the sole required path.

WCAG 2.1.1

CAPTCHA breaks AAA. Use Slate's honeypot/secret-field spam controls — and make honeypots aria-hidden="true" + tabindex="-1" or THEY become the screen-reader trap.

policy

2.4.9 hits hardest in email — "View your application status", never "click here". Avoid pure #000/#fff (auto-inversion); test Outlook + Apple Mail dark.

WCAG 2.4.9

The kit's @media print drops chrome, inks black-on-white, shows link URLs. Receipts get printed — don't print dark backgrounds.

cheap win

Land → orient by landmarks → complete with errors → submit → confirm. Re-test any Slate-rendered form after a Technolutions platform update.

Slate Editor Hints

Accessibility

The patterns that pass

Every control: a non-empty <label for>. Every group: <fieldset><legend>. Errors: aria-invalid + aria-describedby.

Copy-paste

Error field pattern

<div class="wsuf-field wsuf-field--error">
  <label for="zip">ZIP code</label>
  <input id="zip" aria-invalid="true" aria-describedby="zip-e">
  <p class="wsuf-error" id="zip-e">Enter a 5-digit ZIP.</p>
</div>

Error summary (focus on submit)

<div class="wsuf-alert" role="alert" tabindex="-1">
  <h2>There’s a problem</h2>
  <ul><li><a href="#zip">ZIP — required.</a></li></ul>
</div>
