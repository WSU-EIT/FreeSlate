Start Here

No CSS or code knowledge needed. If you can build a form in Slate, you can use this kit. Here's the whole thing in five minutes.

Good news: your form is already styled.

When you add fields in Slate's form builder, they automatically get the WSU look — crimson buttons, proper spacing, accessible labels. You usually don't touch any code at all.

The whole job, in 3 steps

1
Build your form in Slate

Add your questions the normal way in the form builder. Give every field a clear label and mark the required ones.

2
It's already styled

Preview it. It looks like WSU. You didn't have to do anything special — the branding is automatic.

3
Add extras only if you want

Want a colored banner, a highlighted note, or an info box? Copy one of the blocks below into an Instructions field. Optional.

5 golden rules (that's it)

  • Always give every question a label. Slate makes you — just don't leave it blank. The label is what screen-reader users hear.
  • Mark required fields in the builder. The red asterisk and crimson edge appear automatically. Don't type your own asterisk.
  • For birthdays and dates, pick the "Date" field — not the old three-dropdown one (Month / Day / Year). The Date field is cleaner and works for everyone.
  • Don't paste formatted text from Word. It brings invisible junk. Paste as plain text, or use the "paste as plain text" option, then format in Slate.
  • Write link text that makes sense on its own. "Search for your CEEB code" — never "click here". People navigating by links hear only the link text.
