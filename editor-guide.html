Editor Guide

How to build a WSU form in Slate, one task at a time. Plain steps, no code. If you can copy a document and fill in blanks, you can do this.

The big idea: you choose the words and questions. The system makes everything look right — colors, spacing, the WSU look, accessibility. You never pick a font or a color. If you're tempted to style something, you're probably about to make extra work; there's almost always a ready-made block instead.

The whole thing in 5 steps

You don't start from a blank page — you copy a blessed template and change it. That's the secret to never breaking anything.

Make a new form

≈ 2 minutes

  1. In Slate, go to Forms and open the _Templates folder.
  2. Pick the closest one — Event RSVP, Info Request, or Visit Registration — and click Copy Form.
  3. Rename your copy (e.g. "Chemistry Open House RSVP"). Save.
  4. Edit the questions, dates, and thank-you message (the tasks below).
  5. Test the link, then Publish. Done.

Why copy instead of start fresh? The template already has the WSU look, a spam guard, and a friendly thank-you page built in. You get all of that for free and just change the parts that are yours.

Editing your form

Add a question

It's automatically styled — you don't do anything special.

  1. In the form editor, click Insert and pick the kind of question you want: Short answer, Multiple choice, Dropdown, Date, File upload, and so on.
  2. Type a clear label (the question text). Always fill this in — never leave it blank.
  3. For choice questions, type your options one per line.
  4. That's it. Preview and it already looks like WSU.

Make a question required

The red star and crimson edge appear by themselves.

  1. Click the question to open its settings.
  2. Set Required to Yes. Save.
  3. Don't type your own asterisk — Slate adds it, styled correctly, automatically.

Show a question only when…

For example: show "Dietary needs" only when "Staying for lunch?" is Yes.

  1. Add both questions first (the trigger question must come before the one it controls).
  2. Open the question you want to hide/show and find Edit Conditions.
  3. Add a condition that reads like a sentence: Show this when [Staying for lunch?] [is] [Yes]. Save.
  4. Preview and try it — the question appears and disappears as you answer.

Settings that matter for events

Set open/close dates and capacity

  1. Open the form's Properties (or Settings).
  2. Set the open and close dates — the form turns itself on and off.
  3. Set a capacity if there's a limit. When it fills, Slate shows a "full" message automatically.
  4. You can edit the closed / full / deadline messages right there if you want different wording.

Change the thank-you message

  1. In Properties, find the Confirmation page (your copied template already has a friendly one).
  2. Switch the editor to the Source / HTML view before pasting anything (see "Paste text safely").
  3. Change the words. Keep it short and warm — "You're registered! We'll email you the details."

Two habits that prevent 90% of problems

Paste text safely

Word and Google Docs hide invisible junk in copied text. This avoids it.

  1. When pasting from Word/Docs/email, use Paste as plain text (Ctrl+Shift+V / Cmd+Shift+V), or paste into Notepad first, then copy from there.
  2. Then format it in Slate. The WSU fonts and colors take over automatically.
  3. When pasting one of the ready-made blocks from Start Here, always switch the editor to Source / HTML view first.

Test your form before you share it

  1. Open the form's preview link and fill it out like a student would.
  2. Check it on your phone too — that's where most students will see it.
  3. Try leaving a required field blank to confirm the error message appears.
  4. If a conditional question is involved, make sure it shows and hides correctly.

Never do these

  • Don't paste from Word without "paste as plain text" — it brings invisible formatting that breaks the WSU look.
  • Don't edit anything inside the _Templates folder — copy it first, then edit your copy.
  • Don't open the Edit Scripts / Styles area. That's the web team's space; you never need it.
  • Don't try to change colors, fonts, or sizes. The brand decides those — and "our office's blue" is a no (it's a brand rule, not us being difficult).
  • Don't use the old three-dropdown Birthdate. Use the Date field — it's cleaner and works for everyone.

Get help

Stuck, or something looks wrong? That's normal — reach out, don't wrestle with it.
