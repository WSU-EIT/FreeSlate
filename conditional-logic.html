Conditional Logic Theater

Play with this annotated form to see every type of conditional behavior Slate supports. Dashed outlines mark each trigger — answer a question to see what reveals.

⚠
Slate gotcha — how reveals actually work

Slate's show/hide writes style="display:none" directly on .form_question. So a rule like .form_question{display:flex !important} will resurrect hidden conditional fields. Style the children, never force display on the question wrapper. And Slate re-renders fragments via AJAX (country→state swap, conditional pages) — bind JS with event delegation on a stable ancestor or a MutationObserver, never directly to fields that can re-render, or your handlers vanish.

How to use this page

Answer each question below. Dashed-outline annotations explain what Slate condition triggers that reveal. Every pattern here maps to a real Slate "Edit Conditions" rule.

1. Basic show / hide

The simplest conditional: show a field only when a specific answer is selected.

Student type *
Slate condition: show next field when Student type = Transfer (Edit Conditions → field → value equals)

Any student with 27+ credits is considered a Transfer Student at WSU.

2. Cascading reveals

Answer one question to reveal a second; answer that to reveal a third. Common in accommodation and dietary workflows.

Will you or your guests require accommodations? *
Slate condition: show accommodation type when Accommodations = Yes
Which accommodations will you need?
Level 2: checking "Dietary" reveals a specific field. Checking "Other" reveals a text box. Multiple conditions on the same field are ORed together in Slate.

3. Value piping

A value typed into one field appears in a label or instruction further down. Slate calls this a "formula field" or piped default value.

Whatever you type becomes part of the heading below (simulating Slate value piping via {{first_name}})

Great to meet you, future Coug!

Your responses so far will help us personalize your admissions experience.

4. Conditional required

A field that only becomes required when a specific answer was given earlier. In Slate this is "Edit Rules → Required if [condition]".

Are you a Washington state resident?
Slate rule: if Resident = No, make "Home state / country" field required (Edit Rules → Required if)

Slate conditions cheat sheet

Show if
Edit Conditions → Add condition → field equals value. Use for revealing hidden sections.
Hide if
Edit Conditions → Show if → INVERSE of the above. Default-visible, hidden on condition.
Required if
Edit Rules → Required → Add condition. Field becomes required only when condition is true.
Value piping
Field default value formula: {{export_key}} references another field's value. Use in Instructions or label text.
Multi-level
Chain conditions by adding conditions to fields that are themselves conditionally shown. Works but gets complex fast — document your logic.
Accessible rule
Revealed elements must use hidden attribute (not display:none via CSS) so focus management works. Move focus to first revealed field. This is done for you by the kit's JS pattern.
